# 匹配引擎 (Match Engine)

    匹配引擎是任何交易所的核心和大脑。它的唯一功能是高效地撮合买单和卖单。当买方的出价等于或高于卖方的要价时，匹配引擎会立即将这两笔订单匹配并执行，完成一笔交易。所有其他系统，无论是 ViaBTC 的服务器还是 Liquibook，都是基于匹配引擎的功能来构建的。

    在18~19年做交易所的时候，当时这块是深交所20年撮合经验的大佬写的，C++写的，我当时就看了代码而已？？？怎么可能看的到， 当时那套撮合私有化去卖的时候，是200W一套， 香港那边的人买的多，哈哈，懂的都懂。里面很多概念，和算法逻辑我还是记得七七八八的，要让我优化撮合单量突破1W+，其实还好，我们当时单机器支持20W+， 但是我写不出来，哈哈哈。
    
    所以找了几个开源的撮合，看看源码，再看下基于rust去下手，还是说C去下手，哈哈。


# 开源主要如下

## ViaBTC Exchange Server

    ViaBTC Exchange Server 是 ViaBTC 交易所的完整、私有的后台系统。它是一个大而全的解决方案，包含了交易所运作所需的所有功能。

    集成度高：它不仅包括了匹配引擎，还集成了用户管理、账户资金管理、API 接口、市场数据广播、风控系统等所有功能。

    私有技术：它的后台系统是 ViaBTC 团队自主研发和维护的，不对外开放。

    产品化：这是一个可以直接供用户使用的完整产品，而不是一个开发工具。

## BitMEX (git比他EX)

    您提到的 "git比他EX" 极有可能指的是著名的加密货币衍生品交易所 BitMEX。与 ViaBTC 类似，BitMEX 也是一个完整的、私有的交易平台。

    完整交易所：它拥有自己的全套后端系统、用户界面和钱包基础设施。

    衍生品方向：BitMEX 的特点是专注于衍生品交易，特别是永续合约，而非传统的现货交易。

    私有技术：它的匹配引擎和后台系统也是自主开发的，能够满足其高并发、低延迟的交易需求。

## CPPtrader

    CPPtrader 是一个通用术语，通常指任何用 C++ 语言编写的交易系统或交易框架。由于 C++ 具备卓越的性能和低延迟特性，它在高频交易和交易所后端系统开发中非常流行。一个名为 CPPtrader 的项目可能是一个完整的系统，它在内部使用 C++ 来构建所有核心组件，包括其匹配引擎，甚至可能使用了像 Liquibook 这样的开源库。

## Liquibook
    
    Liquibook 是一个用 C++ 编写的、开源的高性能匹配引擎库。它不是一个完整的交易所，而是一个专门用于构建匹配引擎的工具包。

    专一性：它只专注于订单撮合这一件事。它不处理用户账户、资金管理、API 接口或市场数据广播等功能。

    开源性：作为一个开源项目，它允许开发者在其基础上自由地构建自己的交易系统，大大节省了从零开始研发核心引擎的时间和精力。

    高性能：得益于 C++ 的语言特性，Liquibook 能够提供非常低的延迟和高吞吐量，这对于高频交易至关重要。

    可以将 Liquibook 想象成赛车的高性能发动机，而一个完整的交易所则是一辆完整的赛车。
