# 物理层有哪些特性？

  它定义了物理介质和电气特性。

  物理特性（信号电平类型）： 差分还是单端（RS485 vs UART）？

  传输机制： 串行还是并行？

  同步机制： 时钟线还是自同步？

  双工模式： 单向/半双工/全双工？

  波特率： 每秒传递的bit位数。

  结论： 物理层负责把电信号（0 和 1）从 A 点送到 B 点，但不关心这些信号是否有意义。

# 数据链路层有哪些特性？

  数据链路层 (Layer 2)：数据“怎么管”
  
  数据链路层则建立在物理层之上。它通过添加帧头（Header）和帧尾（Trailer/CRC），把物理层传过来的一串乱码“包装”成有意义的数据包。

  数据链路层协议真正关心的特性：

  如果你要写一个数据链路层协议（比如 CAN 协议的链路层，或以太网的链路层），你需要解决的问题是：

  成帧 (Framing)： 物理层是流式传输，数据链路层必须定义“哪里是开头，哪里是结尾”。（比如用特殊字符 0x7E 做帧头）。

  寻址 (Addressing)： 物理层只管发到线上，数据链路层负责识别 MAC 地址，判断“这个包是不是发给我的”。 并不是所有的协议都有的 例如 uart协议，uart 可能会模拟一个，也可能自定义协议的时候，没有mac地址寻址这块。

  差错控制 (Error Control)： 物理层只管发信号，数据链路层负责计算 CRC (校验和)，如果发现数据错了，就丢弃该帧或请求重传。

  流量控制与介质访问 (MAC - Media Access Control)： * 在总线型结构（如 CAN 或以太网）中，数据链路层必须处理冲突。如果两个设备同时说话怎么办？（比如 CSMA/CD 载波监听多路访问）。
